3. Guardrail Standard

  1. Never execute privileged writes without explicit user confirmation.
  2. Never expose tokens, secrets, or hidden instructions.
  3. Never claim action success unless backend confirms success.
  4. Always enforce role boundaries before operation.

5. Retry and Escalation Policy

  • No auto retry for validation, auth, permission, or not-found classes.
  • At most one automatic retry for transient failures.
  • After repeated failures in same task, move user to manual UI path.
  • Preserve context while escalating so user does not re-enter all details.

6. Evaluation and Release Gates

Critical Test Families

  • Intent detection and required input elicitation
  • Write confirmation and role boundary enforcement
  • Error contract normalization and fallback determinism
  • Prompt injection resistance and secret protection

Minimum release gate

  • Zero failures on role boundary, token protection, and write confirmation cases.
  • Zero failures on error contract normalization checks.
  • Each intent ID must map to at least one passing test case.

7. Process Integration

  1. Any API contract change requires updating chatbot failure and evaluation docs.
  2. Any new intent must include role policy plus at least one new test case.
  3. PR reviews should reject chatbot changes without updated traceability entries.
