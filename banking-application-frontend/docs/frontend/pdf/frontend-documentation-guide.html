5. Frontend Operating Standards

Validation

  • Centralized rule set for email, password, name, mobile, confirm-password, and bank fields.
  • Pre-submit validation expected before API invocation.

Error Handling

  • User-safe messaging only; no raw payload leakage in UI.
  • Timeout and retry behavior where allowed by endpoint semantics.
  • Session/auth failures trigger re-authentication guidance.

6. Developer Runbook Summary

  1. Install dependencies in each frontend app folder.
  2. Run development server via npm run dev.
  3. Verify backend ports before end-to-end testing.
  4. Update docs/frontend when route, API, validation, or auth behavior changes.

7. Recommended Governance

  • Require docs update in every PR that changes frontend behavior.
  • Maintain changelog entries for contract, flow, and rule updates.
  • Keep chatbot pack separate but linked for operational clarity.
