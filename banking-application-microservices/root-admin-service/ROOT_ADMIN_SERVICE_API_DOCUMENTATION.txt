ROOT-ADMIN-SERVICE
The Root Admin Service is a backend microservice responsible for root-level administrative actions in the banking application. It manages root admin authentication and the verification of admin users, enforcing strict security and role-based access control for sensitive operations.

Key Features:
- Root admin authentication (sign-in)
- Verification of admin users by root admin
- JWT-based authentication for secure endpoints
- Test endpoint for JWT debugging and validation
- Strict role-based access control (only root-admin can verify admins)

Technology Stack:
- Java 17, Spring Boot 3.x
- Spring Data JPA (MySQL backend)
- Spring Security (JWT)
- Lombok
- Maven

-----------------------------
API ENDPOINTS (Complete List)

1. Root Admin Sign-In
   POST /api/root/signin
   - Authenticates a root admin and returns a JWT token.

2. Verify Admin (Root Admin Only)
   POST /api/root/verify-admin
   - Verifies an admin user using root admin credentials.

3. Test JWT Authentication (Root Admin Only)
   GET /api/root/test-jwt
   - Test endpoint to validate JWT authentication and retrieve current user info.

-----------------------------

DETAILED ENDPOINTS

1. Root Admin Sign-In
Method & Path: POST /api/root/signin
Description: Authenticates a root admin and returns a JWT token.
Request Body (JSON):
  - username (string, required)
  - password (string, required)
Responses:
  Success (200): {"success": true, "message": "Sign-in successful.", "token": "<jwt>"}
  Errors:
    400 Bad Request: {"message": "Invalid credentials."}
    500 Internal Server Error: {"message": "Unexpected error occurred."}

2. Verify Admin (Root Admin Only)
Method & Path: POST /api/root/verify-admin
Description: Verifies an admin user using root admin credentials.
Request Body (JSON):
  - adminUsername (string, required)
  - rootUsername (string, required)
  - rootPassword (string, required)
Responses:
  Success (200): "Admin verified successfully."
  Errors:
    400 Bad Request: {"message": "Invalid credentials or admin not found."}
    403 Forbidden: {"message": "Not authorized."}

3. Test JWT Authentication (Root Admin Only)
Method & Path: GET /api/root/test-jwt
Headers:
  - Authorization: Bearer <token>
Description: Test endpoint to validate JWT authentication and retrieve current user info.
Responses:
  Success (200): {"message": "JWT Authentication successful!", "user": "rootadmin", "roles": ["ROOT_ADMIN"], "timestamp": "..."}
  Errors:
    401 Unauthorized: {"message": "Invalid or missing token."}

-----------------------------
Important Points:
- Only users with ROOT_ADMIN role can access verification and test endpoints.
- JWT authentication is required for all sensitive operations.
- All error responses are JSON with a message field.
- Root admin credentials must be kept secure and rotated regularly.
- This service is critical for the security and integrity of the admin approval workflow.
