

ACCOUNT-Service
The Account Service is a core backend microservice responsible for managing the complete lifecycle of bank accounts in a multi-country, multi-bank environment. It is designed to handle account creation, document management, application status tracking, and administrative approval or rejection workflows for customers in India, USA, and UK. The service enforces strict validation and compliance rules for each country, ensuring that all regulatory and business requirements are met.

Key Features:

- Multi-country support: Implements separate account creation endpoints and request DTOs for India, USA, and UK. Each country has its own validation rules (e.g., Aadhaar/PAN for India, SSN for USA, NIN for UK) and required fields. The service uses a strategy pattern to encapsulate country-specific logic, ensuring compliance with local regulations.

- RESTful API: Provides a set of HTTP endpoints under /api/accounts for all operations, including POST for account creation, document upload, approval/rejection, and GET for status checks, queries, and document retrieval. All endpoints use standard HTTP status codes and JSON responses.

- Multipart file handling: Requires users to upload four mandatory documents (idProof, addressProof, incomeProof, photo) as multipart/form-data during account creation. The service validates presence and type of each file, and stores them securely, associating each with the correct account application.

- Field-level validation: Uses Jakarta Bean Validation annotations (@NotBlank, @Pattern, @Email, @Size, etc.) on DTOs to enforce strict validation for every request field. Validation errors are returned with clear messages specifying the invalid or missing fields.

- Admin workflow: Exposes endpoints for administrators to list pending applications, view details, and approve or reject accounts. Approval updates the account status and assigns an account number. Rejection records the reason and updates the status. All admin actions are auditable.

- Application status tracking: Allows users to check the status of their application at any time using the application ID. The response includes current status, validation results, required documents, and next steps if any.

- Document management: Provides endpoints to upload additional documents, and to download or view documents by document ID. Documents are linked to accounts and validated for type and association.

- Query and reporting: Supports endpoints to retrieve accounts by user ID, bank name, status, or account number. Returns either a list of accounts or a single account, enabling customer support and reporting use cases.

- Extensible design: Uses DTO/entity separation for clean data flow, and a strategy factory for country-specific account creation logic. New countries or account types can be added with minimal changes. The codebase follows Spring Boot conventions for maintainability.

- Security: While authentication and authorization are not implemented in this service, all endpoints are designed for easy integration with security layers (e.g., OAuth2/JWT). Sensitive data is validated and handled according to best practices, and file uploads are restricted to required types.

Technology Stack:
- Java 17, Spring Boot 3.x
- Spring Data JPA (MySQL backend)
- Spring Web & WebFlux (REST APIs)
- Flyway (database migrations)
- Lombok (boilerplate reduction)
- Maven (build)
- SMTP (email notifications, if enabled)

  Important Points:
  - For account creation, all required fields and four documents (idProof, addressProof, incomeProof, photo) must be provided. See country-specific requirements at the end of this file.
  - All files must be PDF, JPG, or PNG and within allowed size limits.
  - Validation errors will specify missing or invalid fields.

  -----------------------------
  Important Points: Required Documents for Account Creation

  For account creation (all countries), the following documents are mandatory and must be uploaded as part of the application:

  - idProof: Government-issued identity proof (e.g., Aadhaar for India, SSN for USA, NIN for UK)
  - addressProof: Proof of address (e.g., utility bill, rental agreement, bank statement)
  - incomeProof: Proof of income (e.g., salary slip, tax return, employment letter)
  - photo: Recent passport-size photograph

  Document requirements by country:

  - **India:**
    - Aadhaar card (12 digits)
    - PAN card (10 characters)
    - Address proof (utility bill, rental agreement, etc.)
    - Income proof (salary slip, tax return, etc.)
    - Passport-size photo

  - **USA:**
    - Social Security Number (SSN, format XXX-XX-XXXX)
    - Address proof (utility bill, lease, etc.)
    - Income proof (pay stub, tax return, etc.)
    - Passport-size photo

  - **UK:**
    - National Insurance Number (NIN, format XX999999X)
    - Address proof (utility bill, council tax bill, etc.)
    - Income proof (payslip, tax return, etc.)
    - Passport-size photo

  All documents must be uploaded in PDF, JPG, or PNG format. Each file must not exceed the maximum allowed size (see error codes for limits). Incomplete or invalid documents will result in application rejection.

  These requirements are strictly enforced for compliance and fraud prevention. Ensure all documents are clear, valid, and up to date before submission.

2. Upload Document
Method & Path: POST /api/accounts/{accountNumber}/documents
Description: Uploads an additional document for the specified account number.
Parameters:
  Path:
    - accountNumber (string, required)
  Query: None
  Headers:
    - Content-Type: multipart/form-data (required)
  Request Body (form fields):
    - file (file, required): The document file to upload (PDF, JPG, PNG only)
    - documentType (string, required): One of ID_PROOF, ADDRESS_PROOF, INCOME_PROOF, PHOTO
Responses:
  Success (201):
    Response Fields:
      - documentId (long): Unique ID of the uploaded document
      - accountNumber (string): Account number associated with the document
      - documentType (string): Type of document uploaded (ID_PROOF, ADDRESS_PROOF, INCOME_PROOF, PHOTO)
      - originalFilename (string): Original name of the uploaded file
      - storedFilename (string): Name of the file as stored on the server
      - fileSize (long): Size of the uploaded file in bytes
      - uploadStatus (string): Status of the upload (e.g., SUCCESS, FAILED)
      - uploadedDate (datetime): Date and time when the document was uploaded
      - message (string): Success message
      - isValid (boolean): Whether the document passed validation
      - validationMessage (string): Validation result or error message (if any)
  Errors:
    400 Bad Request:
      {"message": "Missing or invalid file/documentType."}
    404 Not Found:
      {"message": "Account not found."}
    415 Unsupported Media Type:
      {"message": "Only PDF, JPG, PNG files are allowed."}
    413 Payload Too Large:
      {"message": "File size exceeds limit."}
    500 Internal Server Error:
      {"message": "Unexpected error occurred."}

  Important Points:
  - accountNumber must be valid and exist.
  - Returns all documents for the account or 204 if none.
