

Account Service — Complete & Accurate API Documentation
======================================================

Overview
--------
Service Name: account-service
Base URL: http://localhost:8085
Context Paths: /api/accounts and /accounts (both work identically)
Database: MySQL — accountdb (auto-created)
Port: 8085
Multipart max file size: 10 MB per file, 50 MB per request
Document upload directory: uploads/documents/ (relative to application root)
Email notifications: Gmail SMTP — noreplyinterbankinghub@gmail.com
Admin service dependency: http://localhost:8083

The Account Service handles the full lifecycle of bank account applications: creation with mandatory document uploads, admin review and approval/rejection, and document management. Accounts are never given an account number until an admin explicitly approves them.

------------------------------------------------------


Data Models
-----------

Account Entity (accounts table):
  - id: Long (PK, auto-increment) — Internal identifier
  - userId: String(50) (NOT NULL) — References the user who owns this account
  - accountNumber: String(20) (UNIQUE, nullable) — Generated only after admin approval
  - bank: String(100) (NOT NULL) — Bank name (e.g., "State Bank of India")
  - branch: String(100) (NOT NULL) — Branch name
  - ifscCode: String(15) (nullable) — IFSC / routing code
  - country: String(10) (NOT NULL) — One of: INDIA, USA, UK
  - accountType: Enum(STRING) (NOT NULL) — See AccountType section
  - status: Enum(STRING) (NOT NULL, default PENDING) — PENDING / APPROVED / REJECTED
  - deposit: BigDecimal(15,2) (NOT NULL, >= 0) — Initial deposit amount
  - consent: Boolean (NOT NULL) — Must be true
  - applicationStage: Enum(STRING) (default PERSONAL) — PERSONAL / EDUCATIONAL / INCOME / NOMINEE / DOCUMENTS / COMPLETE
  - personalDetails: (embedded) PersonalDetails — See below
  - educationalDetails: (embedded) EducationalDetails — See below
  - incomeDetails: (embedded) IncomeDetails — See below
  - nomineeDetails: (embedded) NomineeDetails — See below
  - documents: (OneToMany) List<DocumentMetadata> (LAZY) — Associated uploaded documents
  - createdDate: LocalDateTime (auto, not updatable) — Creation timestamp
  - updatedDate: LocalDateTime (auto-updated) — Last update timestamp
  - createdBy: String(50) (nullable) — Who created the record

Indexes: idx_user_id, idx_account_number, idx_status, idx_country_status



AccountStatus Enum:
  - PENDING:   Application submitted, awaiting admin review
  - APPROVED:  Admin approved; account number assigned
  - REJECTED:  Admin rejected the application
  (Only APPROVED accounts have an accountNumber assigned. All others have null.)



AccountType Enum (Country-Specific):
  - SAVINGS:               Savings                 (INDIA, USA, UK)
  - CURRENT:               Current                 (INDIA, UK)
  - SALARY:                Salary                  (INDIA)
  - FIXED_DEPOSIT:         Fixed Deposit           (INDIA)
  - CHECKING:              Checking                (USA)
  - MONEY_MARKET:          Money Market            (USA)
  - CERTIFICATE_OF_DEPOSIT:Certificate of Deposit  (USA)
  - ISA:                   ISA                     (UK)
  - FIXED_TERM:            Fixed Term              (UK)
  (The valid set is enforced by country. Using an invalid type for a country returns a 400 error.)



ApplicationStage Enum:
  - PERSONAL:    Personal Details   (Stage 1 — personal info)
  - EDUCATIONAL: Educational Details(Stage 2 — education info)
  - INCOME:      Income Details     (Stage 3 — income info)
  - NOMINEE:     Nominee Details    (Stage 4 — nominee info)
  - DOCUMENTS:   Documents          (Stage 5 — document upload)
  - COMPLETE:    Complete           (All stages done; account is ready for admin review)
  (The service enforces stage progression and validation at each step.)


PersonalDetails (Embedded):
  - fullName:      String(100) (NOT NULL, 2–100 chars) — Full legal name
  - dateOfBirth:   LocalDate (NOT NULL, must be in past) — Date of birth
  - gender:        Enum(STRING) (NOT NULL) — Male / Female / Other
  - email:         String(100) (NOT NULL, valid email) — Contact email
  - address:       TEXT (NOT NULL, 10–500 chars) — Residential address
  - aadhaar:       String(12) (nullable, 12 digits) — India only
  - pan:           String(10) (nullable, pattern [A-Z]{5}[0-9]{4}[A-Z]) — India only
  - mobile:        String(15) (nullable, pattern ^[6-9][0-9]{9}$) — India only (10-digit Indian mobile)
  - ssn:           String(11) (nullable, pattern ^[0-9]{3}-[0-9]{2}-[0-9]{4}$) — USA only
  - phone:         String(15) (nullable, pattern ^\+1[0-9]{10}$) — USA only
  - nin:           String(9) (nullable, pattern ^[A-Z]{2}[0-9]{6}[A-Z]$) — UK only


EducationalDetails (Embedded):
  - educationLevel:    Enum(STRING) (NOT NULL) — See values below
  - institutionName:   String(200) (NOT NULL, 2–200 chars) — School/college/university name
  - course:            String(150) (NOT NULL, 2–150 chars) — Name of course or degree
  - yearOfCompletion:  Integer (NOT NULL, 1950–2030) — Year course was completed
  - grade:             String(20) (optional) — Grade or percentage

EducationLevel values:
  - India & UK: High School, Diploma, Graduate, Post Graduate, Professional
  - USA: Associate Degree, Bachelor Degree, Master Degree, Doctoral Degree
  - UK: GCSE, A-Levels, Undergraduate Degree, Postgraduate Degree, Professional Qualification


IncomeDetails (Embedded):
  - employmentStatus: Enum(STRING) (NOT NULL) — Employed / Self-Employed / Business / Business Owner / Student / Retired / Unemployed
  - employerName:     String(200) (optional) — Employer name if employed
  - occupation:       String(100) (NOT NULL, 2–100 chars) — Job title or occupation
  - monthlyIncome:    BigDecimal(15,2) (NOT NULL, >= 0) — Monthly income in local currency
  - annualIncome:     BigDecimal(15,2) (NOT NULL, >= 0) — Annual income
  - incomeSource:     Enum(STRING) (NOT NULL) — Salary / Business / Investment / Pension / Social Security / Other
  - currencyCode:     String(3) (NOT NULL, exactly 3 chars) — INR (India), USD (USA), GBP (UK)


NomineeDetails (Embedded):
  - nomineeName:      String(100) (NOT NULL, 2–100 chars) — Full name of nominee
  - nomineeRelation:  Enum(STRING) (NOT NULL) — Spouse / Father / Mother / Son / Daughter / Brother / Sister / Other
  - nomineeDob:       LocalDate (NOT NULL, must be in past) — Nominee's date of birth
  - nomineeContact:   String(15) (NOT NULL, pattern ^[+]?[0-9]{10,15}$) — Nominee's phone number
  - nomineeAddress:   TEXT (NOT NULL, 10–500 chars) — Nominee's address


DocumentMetadata Entity (document_metadata table):
  - id: Long (PK, auto-increment) — Internal identifier
  - accountId: Long (NOT NULL) — Foreign key to accounts.id
  - documentType: Enum(STRING) (NOT NULL) — ID_PROOF / ADDRESS_PROOF / INCOME_PROOF / PHOTO
  - originalFilename: String(255) (NOT NULL) — Original name of uploaded file
  - storedFilename: String(255) (NOT NULL) — Name stored on disk
  - filePath: String(500) (NOT NULL) — Full disk path
  - fileSize: Long (NOT NULL, >= 1) — Size in bytes
  - contentType: String(100) (NOT NULL) — MIME type (e.g., application/pdf, image/jpeg)
  - uploadStatus: Enum(STRING) (default UPLOADED) — UPLOADED / VERIFIED / REJECTED
  - uploadedDate: LocalDateTime (auto) — Upload timestamp


AccountCreationResponse (Response DTO):
  - accountNumber: null (until admin approval)
  - applicationId: string (always present — use for tracking)
  - userId: string
  - status: PENDING
  - country: INDIA
  - createdDate: 2026-04-24T10:30:00
  - validationResults: (present only if validation errors)
      PERSONAL: isValid: false, errors: [...], warnings: [...]
  - requiredDocuments: [ID_PROOF, ADDRESS_PROOF, INCOME_PROOF, PHOTO]
  - applicationStage: Complete
  - completionPercentage: 100.0
  - message: Application completed successfully...
  - warnings: []
  - nextSteps: [Wait for admin approval, Check application status using your Application ID, ...]

### 1. Create India Account

**POST** `/api/accounts/create/india`

**Content-Type:** `multipart/form-data`

**Description:** Creates a new bank account application for an Indian customer. All four documents are **mandatory**. Account is placed in PENDING status and requires admin approval before an account number is issued.


Form Fields (all required unless noted):
  userId: String (NOT BLANK) — User ID from user-service
  bank: String (NOT BLANK) — Bank name
  branch: String (NOT BLANK) — Branch name
  ifscCode: String (optional) — IFSC code
  fullName: String (2–100 chars) — Full legal name
  dob: LocalDate (format yyyy-MM-dd) — Date of birth
  gender: String (Male/Female/Other) — Gender
  mobile: String (^[6-9][0-9]{9}$) — 10-digit Indian mobile
  email: String (valid email) — Email address
  address: String (min 10 chars) — Residential address
  aadhaar: String (^[0-9]{12}$) — 12-digit Aadhaar number
  pan: String (^[A-Z]{5}[0-9]{4}[A-Z]{1}$) — PAN card number
  educationLevel: String (High School/Diploma/Graduate/Post Graduate/Professional) — Education level
  institutionName: String (NOT BLANK) — Institution attended
  course: String (NOT BLANK) — Course/degree name
  yearOfCompletion: Integer (1950–2030) — Year of graduation
  grade: String (optional) — Grade or percentage
  employmentStatus: String (Employed/Self-Employed/Business/Student/Retired/Unemployed) — Employment status
  employerName: String (optional) — Employer name
  occupation: String (NOT BLANK) — Job title
  monthlyIncome: BigDecimal (>= 0) — Monthly income
  annualIncome: BigDecimal (>= 0) — Annual income
  incomeSource: String (Salary/Business/Investment/Pension/Other) — Source of income
  nomineeName: String (NOT BLANK) — Nominee full name
  nomineeRelation: String (Spouse/Father/Mother/Son/Daughter/Brother/Sister/Other) — Relation to nominee
  nomineeDob: LocalDate (format yyyy-MM-dd) — Nominee date of birth
  nomineeContact: String (^[0-9]{10}$) — Nominee phone number
  nomineeAddress: String (NOT BLANK) — Nominee address
  accountType: Enum (SAVINGS/CURRENT/SALARY/FIXED_DEPOSIT) — Account type for India
  deposit: BigDecimal (>= 0) — Initial deposit amount
  consent: Boolean (must be true) — Terms consent


File Parameters (all required):
  idProof: Aadhaar card, Passport, Voter ID (PDF, JPEG, PNG, max 10MB)
  addressProof: Utility bill, Bank statement, Lease agreement (PDF, JPEG, PNG, max 10MB)
  incomeProof: Salary slip, ITR, Bank statement (PDF, JPEG, PNG, max 10MB)
  photo: Passport-size photograph (JPEG, PNG, max 10MB)

  message: Application completed successfully with all documents uploaded. Your application ID is APP-42. Now pending admin verification and approval.
  requiredDocuments: [ID_PROOF, ADDRESS_PROOF, INCOME_PROOF, PHOTO]
  nextSteps: [Wait for admin approval, Check application status using your Application ID, Admin will review your documents and approve, Account number will be provided after approval]



Validation Error Response — HTTP 400 Bad Request:

    EDUCATIONAL: isValid: true, errors: [], warnings: [Year of completion is in the future]

## Security & Validation

- All endpoints require authentication (JWT recommended in production).
- Strict validation on all fields (see Data Models for constraints).
- CORS is enabled for trusted frontend origins only.
- File upload size and type are enforced at both API and server config.
- Only admins can approve/reject accounts or access admin endpoints.

## Integration Points

- User Service: For user identity and userId reference.
- Admin Service: For admin review and approval workflow.
- Email Notification Service: Sends notifications on application status changes.

## Additional Business Rules

1. All four documents are required at creation; missing any returns a 400 error.
2. Application ID is always returned on creation for tracking.
3. Account number is generated only after admin approval.
4. Document status is tracked (UPLOADED, VERIFIED, REJECTED).
5. Duplicate accounts (same user, bank, branch) are not allowed.
6. Enum values are case-insensitive but must be valid for the country.
7. All error responses are JSON except for plain text in some legacy admin endpoints.

## Error Handling (Expanded)

| HTTP Code | Scenario |
|---|---|

| 204 | Empty result (no records found) |
| 400 | Validation failure, duplicate account, invalid enum value, missing required field, invalid document type |

Business Rules
--------------
1. Account number is NOT generated at creation. It is only generated upon admin approval.
2. Duplicate check: A user cannot have more than one account in the same bank+branch combination.
3. All four documents (ID Proof, Address Proof, Income Proof, Photo) are mandatory at account creation. The old JSON-only endpoint (POST /api/accounts/create/{country}) is disabled and returns a 400 with guidance.
4. Application ID format: APP-{internal_database_id} — always return this to the user immediately after creation for tracking.
5. Email notification: On successful application submission, EmailNotificationService.notifyAdminAccountCreated() is called to notify relevant admins.
6. Account number format (on approval): Three-character country prefix (IND, USA, GBP) + date (YYYYMMDD) + sequential number, e.g., IND20260424001.
